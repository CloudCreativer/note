## 一、開場：鑽石組與無為而至

### 鑽石故事

- 鑽石進口商：完整度高的鑽石**一顆一顆放**；碎鑽**一包一包放**——因為**鑽石會刮傷鑽石**。
- **隱喻**：被稱為「鑽石組」的隊伍，成員互相刮傷、搶對話權，討論不出結果。
- 過去習慣：話很多、要彰顯「我講的才是對的」→ 圖完成度高，但**不是做局**。

### 無為而至（本課核心練習）

- 不是用「大愛用說話」來完成事情。
- 要練的是：**不要說話，但面前的局能夠被完成**。

## 二、做事 vs 做局：279 間公司

### 詹宏志的連環問

| 規模 | 直覺反應 | 問題 |
|------|----------|------|
| 1 間 | 親自管、親自幹 | — |
| 3 間 | 努力 ×3 | 還是「自己管」思維 |
| 7 間 | 更努力、每天管 | 同上 |
| **279 間** | 不可能親自管 | 才想到：每間找總經理、建匯報 |

- 279 間 × 每年股東會 + 董事會 = 365 天幾乎都在開會 → **物理上不可能親自管**。
- **關鍵問題**：為什麼現在只有 1 間、你覺得管得來時，不會這樣做？

### 做事 vs 做局的差別

| 做事 | 做局 |
|------|------|
| 總覺得面前的事只有你能做 | 設計機器／系統替你把事完成 |
| 放開會覺得自己「沒用、最廢」 | 打造持續運作的局，人去度假 |
| 創造本不需要的位子讓自己看起來有用 | 對的人 + 對的位置 + 對的流程 |
| 279 間不可能用同一套「親自管」 | 1 間管得好 → 279 × 1 的複製邏輯 |

> 如果你請一個人幫你管 279 間，你一定看他**管那一間時怎麼管**——因為那一間管得好，剩下就是 279 × 1。

## 三、會議四角色

| 角色 | 功能 | 常見問題 |
|------|------|----------|
| **發言者** | 全組講完後，代表團隊對外發言 | 容易搶戲、個人英雄 |
| **筆記者** | 統計、記錄眾人想法，會後匯報分工 | **Kim 認定最關鍵** |
| **吹哨者** | 覺得討論效率差、方向歪了要舉手 | 現場多組舉手，但平時不說 |
| **時間守門員** | 控時間、加權比重 | 常被忽略 |

### 為什麼筆記者最關鍵？

1. **局自己**：舉手說「我要當筆記者」→ 沒人阻止 → 你把自己局在「只負責記錄」。
2. **輸出後大家會忘**：成員依賴筆記者的彙整 → 筆記者可影響派系、分配任務。
3. **匯報時不點自己**：偉大 CEO 的修煉起點——Tim Cook 就是筆記者出身。
4. **Kim 的建議**：**從今天起，進任何會議優先搶筆記者位置**。

### 加權比重（決策 vs 專業）

- 老闆在**決策**上加權高 ≠ 在**專家領域**上加權高。
- 例：修車——技師在「怎麼修」上發言權 > 老闆；老闆在「要不要修、花多少」上決策權較大。

### 時間守門員的失敗模式

- 40 分鐘 5 題 → 應平均或依權重分配（標準最重要 → 最多時間）。
- 常見錯誤：大家輪流發言 → 統計完才發現**時間已耗盡** → 剩 5 分鐘沒意義。
- **先定規則再討論**：愛因斯坦邏輯——99% 時間想框架，1% 解問題。

## 四、共識 vs 多數決

- **不要用多數決**：看似公平，實則**霸凌**——永遠有少數被碾過。
- 投票設計有 17～30 種（二分之一、三分之二、四分之三、選舉人團等）→ 讀完規則才知誰吃虧。
- **能不用投票就不用** → 採取**共識**（consensus）。

## 五、程式（成事）的順序

### 5.1 討論前的 meta 規則

- 上一局若用混亂方式討論，這一局**不可能更有效率**。
- 必須先建立：**我們要如何討論？** 再開始討論內容。
- 避免：每人各想各的 → 筆記彙整 → 開始投票 → 變成哄搶。

### 5.2 簡報框架（四步驟 × 四問題）

與「成事的四個標準」直接對應；每發生一個步驟，問一個關鍵問題，才進下一步。

| 步驟 | 內容（各組彙整 + Kim 確認） | 關鍵問題 |
|------|------------------------------|----------|
| **1** | 確認彼此的目標與利益 | 在這件事上，**你想要得到什麼**？／每個人想要什麼？ |
| **2** | 角色匹配：盤點能力，確認是否在適合的位置 | 你**擅長**做什麼？不擅長／不適合時，我**怎麼協助**？ |
| **3** | 溝通協調：位置不夠、意願不足時的交換 | **我能做什麼，讓你願意去做這件事**？／可以跟你交換什麼？ |
| **4** | 價值分配機制：確保每人得到在意的價值 | 你有信心完成嗎？完成後**希望獲得什麼回報**？／標準對你是否合理？ |

**步驟 2 的細化邏輯**（遊樂園／專案比喻）：

1. 先問每人想從這件事得到什麼（錢、舞台、學習…）。
2. 再問是否在**適合且願意**的位置——團隊常無法讓每人都如願 → 進入步驟 3 交換。
3. 步驟 4 可含：設 SOP／流程標準、個人 KPI（非只有大 KPI），確保各位置發揮最大效益。

### 5.3 Kim 官方版四步驟（講師總結）

```
① 找到最大目標／光因素（所有人都看見同一目標）
   └ 問：什麼原因讓你不想做？→ 處理「反對這個共同目標」的人

② 取得共識
   └ 問：我給你什麼，你會願意做這件事？

③ 分工（持續處理意願問題）
   └ 問：做什麼事會讓你表現最好？

④ 若不符合預期
   └ 咒語：我能提供什麼，願意讓你做這件事？
```

→ 四步循環，可解決面前很多問題。

### 5.4 神奇咒語

> **「我能提供什麼，願意讓你做這件事？」**

| 場景 | 用法 |
|------|------|
| 殺價／買房買車 | 我能做什麼，買到更便宜的？ |
| 加薪 | 老闆，我做什麼你可以讓我加薪？ |
| 客戶不買 | 我提供什麼，你願意買？ |
| 團隊協作 | 我能做什麼，讓你願意接這個角色？ |

**要經常、勇於說這句話**——這是做局的核心工具之一。

### 5.5 各組報告摘要

| 組別 | 特色 |
|------|------|
| A 組 | 四步驟連線完成；步驟 4 含 SOP／個人 KPI |
| B 組 | 教室專配：了解特性＋專業（MBTI 等）；第三題因未達共識較難收斂 |
| D/W 組 | 目標對齊 → 能力資源 → 異議納入 → 價值滿足 |
| JH5 組 | 用自己的「開有限公司」框架：共識 → 資源 → 流程 → 問題被解決 → 循環回共識 |

## 六、布局心法

### 不要注入自己的想法

- 假開放：「我吃什麼都可以啦，**不要咖哩飯**就好」→ 其實有偏好，卻偽裝沒有。
- 引導別人用**你的**解決方案 → 也是注入。

### 不舉 · 四個「對」

布局裡有四個東西**必須是對的**，錯一個後面不用看：

| # | 要素 | 錯了會怎樣 |
|---|------|------------|
| 1 | **對的人** | 選錯候選人 |
| 2 | **對的位置** | 擺錯主選位置 |
| 3 | **對的目標** | 拼錯 KPI |
| 4 | **對的路** | 方向錯，全反射 |

→ 與「公司組成／選舉／管委會」邏輯相同。

### 成群 vs 結隊

| 成群 | 結隊 |
|------|------|
| 把人聚集在一起 | 能把事情完成 |
| 聚在一起但解決不了問題 = **沒意義** | 成群 **≠** 結隊 |

**要學會**：既能把人聚在一起，又能解決問題。

## 七、讓人願意留下：四層次 × 三感

### 四個不同層次（不是同一件事）

```
願意進來 → 願意參與 → 願意付出 → 願意留下
```

- 進來 ≠ 參與（有人只來開會、毫無作為）
- 參與 ≠ 付出
- 付出 ≠ 留下（優秀員工付出到一半離開）
- **Google 案例**：開除高薪工程師的方式——叫來公司但**不給專案**，不准領工作做 → 失去意義感 → 自己辭職。

### 三感 + 語言線索

| 感覺 | 說明 | 指標 |
|------|------|------|
| **意義感** | 我為什麼還要在這件事上付出？ | 沒意義感 → 不會復甦 |
| **安全感** | 不符合可以講；別人會理解 | 語言從「我」→「**我們**」（歐巴馬勝選演講） |
| **歸屬感** | 這群人有没有「不服輸／我們是一夥的」 | — |

第二輪討論時，學員反映**更容易在團隊表達意見** → 安全感出現的結果。

## 八、做局的結構：選舉模型

從小型模型看世界的運作邏輯（拉麵社、火星計畫群、社團、公司皆同）：

| 順序 | 動作 | 說明 |
|------|------|------|
| 1 | **設立召喚點** | 選舉成立辦事處 |
| 2 | **創造共同話題** | 常見：找共同討厭的對象（派系 A vs B vs C）→ 不一定真支持候選人，但討厭同一邊 |
| 3 | **設計互動機制** | 極簡也可（按讚、掌聲）→ 產生「我們有共同語言」 |
| 4 | **建立核心領導** | 核心領導出現 |

## 九、Same page 與眾人之力

### Same page（同一平面）

- 討論品質好的人：**資訊流動順**——講話沒人聽不懂，全員討論同一件事。
- 品質差的隊伍：有人不在同一平面、接到球不傳、流動慢 → **光看就知道哪組差**。

### 核心心法

| ❌ 錯誤 | ✅ 正確 |
|---------|---------|
| 用**個人之力**解決**眾人**問題 | 用**眾人之力**解決**個人**問題 |
| 覺得自己最厲害 → 問題全丟給你 | 意識到「我不厲害」→ 大家願意幫你 |
| 辦公室變諮商所，全是問題 | 大家帶來「我願意提供 XXX 幫助」 |

### 成群 vs 絕對（做局兩階段）

| 成群 | 絕對 |
|------|------|
| **前置工作** | **落局核心** |
| 讓人進來、產生連結、形成可運作群體 | 讓這件事**能被完成** |
| 程序：把人聚在一起 | 絕對：把事做完（透過布局讓人願意留下、貢獻） |

## 十、兼容 vs 包容

| | 兼容 | 包容 |
|---|------|------|
| **定義** | 我只要你**好的那一部分**，其他不管 | 你幹啥我都可以接 |
| **比喻** | NBA：只要你贏球，私德不管 | 廟裡拜拜：好消息會去還願，因為上次壞消息被解決了 |
| **策略效果** | 只取所需 | 所有災難會朝你湧來（Kim 某年 1700+ 封自殺相關信） |
| **Kim 建議** | 可作**人生主策略** | **可練習**，練久了格局提升；作主策略會很累 |

**包容的練習價值**：

- 大飄 LED 黑屏踩線事故 → 不當場罵，問「下次會不會進步？」→ 團隊自己成長。
- 婚禮蛋糕砸新娘 → 新人大笑拍照：「這是唯一一次這畫面」。
- **注意**：過度攔火 = 扼殺對方練習包容、學習成長的機會。

> 相同的東西裡找不一樣的；不一樣的東西裡找相同的——這場很難用簡報完全呈現，**練過才看得懂**。

## 十一、休息前練習：保健品公司

| 規則 | 說明 |

**限制**：天馬行空可以，但不要包 Kim 的腦、不要他的名字、不要用保方的產品名稱。
