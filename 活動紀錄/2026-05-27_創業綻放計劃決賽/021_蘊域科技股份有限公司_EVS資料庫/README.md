# 021 蘊域科技 · EVS 跌倒偵測

## 30 秒看懂

蘊域科技（簡報品牌 `YUN X`）以 **EVS（事件式視覺）** 做跌倒與異常行為偵測：只記錄畫面變化而非連續錄影，主打隱私友善。  
商業路徑是「硬體先落地、軟體再擴張」：先以裝置導入醫療/長照場域，後續以 SDK 授權擴到更多設備與通路。

---

## 他們在做什麼

| 面向 | 說明 |
|------|------|
| **技術** | EVS 事件式感測（合作夥伴含 **Prophesee**、聯發科等），以「事件變化」取代連續影像串流 |
| **產品** | 簡報可見 `VIGIL X` 裝置與軟體能力，並規劃 SDK 對外授權 |
| **場景** | 長照、醫療機構、居家（像 iCam 安裝） |
| **資料** | 團隊主張已累積多年 EVS 人類行為資料，作為模型與場域調校基礎 |

---

## 解決什麼問題

- 長照／醫療：傳統攝影隱私疑慮、雷達易受環境干擾；穿戴式老人不願戴。  
- EVS：用事件變化資料做行為辨識；簡報主張反應速度快、可支援即時告警。  
- 機構需即時通報；居家需 Wi‑Fi 通報多台手機。

---

```text
Phase 1：裝置落地機構場域，累積資料與導入案例

```

| 收入 | 說明 |
|------|------|

**募資用途（簡報）：** 次世代裝置、核心專利、晶片量產、海外展會等。

### 通報場景（Q&A）

- **居家：** 裝機 → Wi‑Fi 對接手機，一台通報多台。  
- **機構：** 納入照護系統（MQTT 等）。  
- **新光：** 納入 SKS 系統。

### 1 ms 說明（Q&A）

- 技術上幀率可達極高（現場稱最高約 **10,000 fps** 量級）；跌倒場景實際約 **62 fps**。

## 評審提問與回答

### Q1. 專利不好申請，很多單位要用會不會很快有競爭者？

團隊回覆重點是「資料庫與場域經驗」：已累積多年 EVS 人類行為資料，並以演算法與落地整合作為差異化。對於大廠競爭，團隊認為從合作談判到產品化仍有時間差。

### Q2. 賽道領先者之外，還有沒有其他競爭者？

團隊提到 Sony、Prophesee 主要在感測與硬體面向，蘊域則強調應用層與場域演算法落地。

### Q3. 有鎖定個人家用或機構市場嗎？

全台醫療／長照示範與出貨約 **50+**；產品可裝居家，亦主攻機構。

### Q4. 物理感測是用 CMOS 還是不同原理？

回覆是「非傳統連續影像串流」：以事件變化資料進行辨識，降低完整影像紀錄的隱私負擔。

### Q5. 能否用 30 秒說明從跌倒到通報的場景？

居家：像 iCam 裝機、Wi‑Fi 對接手機即可；機構：訊號納入照護系統；新光：納入 SKS。

### Q6. 行政管理、人員配置與法遵？

曾發生勞健保轉換過程錯誤，**已釐清**（現場簡短說明）。

### Q7. 反應速度 1 ms 如何定義？

團隊說明「硬體能力上限」與「實際場景設定」不同：跌倒偵測會採用較符合應用需求的運行參數，不直接等同理論最高速度。

## 我學到的 3 件事

1. **隱私友善是 EVS 的第一個商業切角**：先解決攝影監控在長照/醫療場域的隱私障礙，才能進入部署決策。  
2. **資料庫與場域部署是護城河核心**：硬體感測器可被替代，但長期資料與客製化演算法更難複製。  
3. **B2B 策略會從硬體走向授權**：先靠裝置落地拿到資料與客戶，再以 SDK 擴張到更多設備與通路。
