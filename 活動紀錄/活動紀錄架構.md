# 活動紀錄架構

> 講座、論壇、工作坊、創業活動 — 把「有去」變成「帶得走」

---

## 為什麼需要這套架構

活動結束一週後，真正會再翻開的通常不是逐字稿，而是：

| 順序 | 區塊 | 回答什麼 |
|------|------|----------|
| 1 | 30 秒看懂 | 這場活動在講什麼？ |
| 2 | 章節切分 | 各段講者／議題與重點 |
| 3 | 重點摘要 | 全場 TL;DR、可對外轉述 |
| 4 | 知識卡片 | 可複用的框架、原則、工具 |
| 5 | 學到什麼 + Action | 我接下來要做什麼 |

**會議記錄** 偏重決議與分工；**活動紀錄** 偏重輸入、講者觀點與知識沉澱。  
若有錄音／VTT，建議固定走下方 **五階 Pipeline**，品質才一致。

---

## 與其他筆記的差異

| 類型 | 路徑 | 核心問題 | 典型來源 |
|------|------|----------|----------|
| **活動紀錄** | `活動紀錄/` | 講了什麼？我學到什麼？ | 論壇、Demo Day、工作坊、交流會 |
| **會議記錄** | `會議記錄/` | 決定了什麼？誰去做？ | 內部會議、客戶會議、籌備會 |
| **課程筆記** | `線上與實體課程/` | 怎麼學會？明天做什麼？ | 系列課、培訓、講座課程 |

同一場活動若**同時有決議**（例如籌備會），可另開一篇 `會議記錄/`；對外參加則寫在 `活動紀錄/`。

---

## 資料夾結構

```text
活動紀錄/
  活動紀錄架構.md          ← 本檔（範本與規範）
  YYYYMMDD_活動名稱_場次/
    README.md              ← 主筆記（必備）
    assets/                ← 選用：簡報、照片、截圖
```

### 命名規則

| 項目 | 格式 | 範例 |
|------|------|------|
| 資料夾 | `YYYYMMDD_活動名稱_場次或關鍵字` | `20260603_AppWorks_AI_Founder_Day_CTO` |
| 主檔 | `README.md` | — |
| 逐字稿（放 repo 外亦可） | 與活動同日、可辨識場次 | `20260603_AppWorks AI Founder Day_1.vtt` |

- 同一活動多場次（CEO / CTO）：**分資料夾**，勿混在一篇。
- 同一場次多份 VTT：在 frontmatter `source` 列清，正文註明各份涵蓋時段。

---

## 五階 Pipeline（有逐字稿時必走）

```
原始素材（VTT / 錄音轉稿 / 手寫速記）
        ↓
① AI 清理      校正專有名詞、講者、重複句、明顯 ASR 錯字
        ↓
② 章節切分      依時間軸或議程分段，每章標講者與重點
        ↓
③ 重點摘要      全場 TL;DR + 各章精華（非逐字稿）
        ↓
④ 知識卡片      可複用框架、對照表、原則（獨立小卡）
        ↓
⑤ 寫入 MD       產出 README.md，補連結、Action、待確認
```

### 各階要做什麼

| 階段 | 產出 | 不要做的事 |
|------|------|------------|
| **AI 清理** | 可讀的段落、講者標籤 | 保留滿篇錯字當正文 |
| **章節切分** | 第 N 章｜標題（起迄時間） | 整篇不分段 |
| **重點摘要** | 30 秒看懂、全場對照表 | 貼逐字稿 |
| **知識卡片** | 3–8 張獨立小卡 | 只有 bullet 沒標題 |
| **寫 MD** | 完整 frontmatter + 區塊 | 缺來源與活動連結 |

### 有現場簡報照片時

1. PNG/HEIC → **`assets/images/jpeg/`**（建議 `sips -s format jpeg`）
2. 新增 **`assets/images/照片索引.md`**（依講者／SECTION 分類）
3. README 內用相對路徑嵌入關鍵圖：`![](assets/images/jpeg/IMG_xxxx.jpg)`
4. frontmatter：`has_images: true`；`source` 列出 VTT + 照片資料夾
5. 逐字稿未齊時：用「素材涵蓋範圍」表標 ✅/❌，`status: draft`

## 完整版區塊一覽

適合：有逐字稿或 1hr+、多講者、想當知識庫的活動。

| 區塊 | 回答什麼 |
|------|----------|
| 活動資訊 | 主辦、日期、地點、官方連結、本筆記涵蓋範圍 |
| 30 秒看懂 | 一句話 + 可選三層結構 |
| 處理紀錄（Pipeline） | 五階流程圖（可簡短） |
| 章節切分 | 各章：講者、時間、重點、表格 |
| 重點摘要（全場） | TL;DR、主線對照、與外部文章對照 |
| 知識卡片 | 獨立標題、圖示／表格、可複用 |
| 金句與現場梗 | 選記，方便回顧 |
| 連結與資源 | QR、簡報、講者文章 |
| ✅ Action Items | 我（或團隊）會做的下一步 |
| ⚠️ 待確認 | 逐字稿缺段、數據待查 |
| 📈 本次學到什麼 | 認知更新（至少 3 條） |
| 更新紀錄 | 版本與日期 |
| 簡報與現場照片 | JPEG 路徑 + 照片索引（可選） |

## 填寫檢查清單

活動結束後 **當天或隔日** 填完：

- [ ] frontmatter 的 `type` 是 `event-notes`（不是 `meeting-notes`）
- [ ] **活動資訊** 含官方連結與「本筆記涵蓋範圍」
- [ ] **30 秒看懂** 能在電梯裡講完
- [ ] 有 VTT 時已走 **五階 Pipeline**，正文非逐字稿
- [ ] **章節** 每段有講者（或標「主辦／未知」）與重點
- [ ] **知識卡片** 至少 3 張，且標題可搜尋
- [ ] **學到什麼** 至少 3 條，含「可複用到別處」的洞察
- [ ] **待確認** 列出缺段、未查證數據
- [ ] **Action** 有具體下一步（可全是「自己」）

若 **30 秒看懂** 寫不出來 → 可能只聽了片段，需補逐字稿或重聽。  
若 **知識卡片** 寫不出來 → 可能停留在資訊瀏覽，試問「哪個框架能帶回公司用？」

## 與其他規範的關係

- 本架構專用於 **對外參加或主辦的活動** 筆記
- **會議記錄** 見 `會議記錄/會議記錄架構.md`（決議、分工）
- **系列課程** 見 `線上與實體課程/學習筆記架構.md`（Input / Insight / Action）
- 附件多時使用 `assets/`；公司 Pitch 類深度整理可參考 `創業綻放計劃決賽/NOTE_STANDARD.md`
