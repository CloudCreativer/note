# 002 皓準科技股份有限公司 自行車齒輪創新

## 一句話摘要

皓準科技股份有限公司的主題聚焦於自行車齒輪鐵片創新，核心賣點包含透過人體數據分析進行客製化設計，以及使用 AI 分析提升騎乘效率，目前重點在釐清成效數據、收費模式與市場營運狀況。

## 背景

- 主題公司：皓準科技股份有限公司
- 主題方向：自行車齒輪鐵片創新
- 已知切入點包含人體數據分析、客製化設計、AI 分析
- 現階段目標是建立產品理解，並整理商業模式與市場驗證相關問題

## 重點整理

### 產品與技術方向

- 自行車齒輪鐵片創新
- 分析人體數據後進行客製化
- 使用 AI 進行分析

### 初步觀察

- 產品價值主張聚焦在提升騎乘效率與個人化適配
- 若客製化建立在個人數據上，後續可擴張性與標準化流程會是關鍵問題
- 若主打效率提升 2.7%，需要釐清其量測方式、樣本數與商業說服力
- 商業化過程中若出現指標從 82 降到 70 且營收下降，代表市場轉換或產品定位可能存在落差

## 關鍵結論

- 這份筆記目前屬於產品定位與商業驗證問題整理階段
- 後續應優先補齊效率提升的量化依據、收費模式與市場營運數據
- 客製化如果依賴非通用數據，需特別釐清規模化能力與持續適配機制

## Q&A

### 效率提升 2.7% 與收費模式

- 問題：提升效率 2.7% 的依據是什麼？收費模式如何設計？
- 可追問方向：
  - 2.7% 是以功率、速度、踩踏效率、疲勞降低，還是比賽成績作為衡量？
  - 測試對象數量、測試條件、對照組與統計方式為何？
  - 收費是一次性產品銷售、客製化服務費、訂閱制，還是搭配檢測服務？

### ToB 與海外市場營運

- 問題：ToB、美國市場與海外市場的營運狀況如何？
- 可追問方向：
  - 主要客群是品牌商、車隊、經銷商、訓練單位，還是終端騎士？
  - 美國市場目前是試點、正式銷售，還是經銷合作階段？
  - 海外市場的營收占比、主要地區與通路策略為何？

### 客製化數據非通用數據如何規劃

- 問題：客製化數據若屬於非通用數據，後續如何規劃？
- 可追問方向：
  - 個人資料如何沉澱成可重用的模型或產品規格？
  - 客製化流程是否能模組化，降低每位使用者的導入成本？
  - 是否有標準資料欄位、分群方法或推薦邏輯，以支持規模化？

### 使用者如何知道騎乘效率更好

- 問題：使用者如何知道自己的騎乘效率確實變得更好？
- 可追問方向：
  - 是否有前後測報告、App 儀表板、功率數據或體感回饋機制？
  - 是否提供試用、測試流程或第三方驗證方式？
  - 對一般消費者與專業選手，成效證明方法是否不同？

### 為什麼步入市場後反而 82 到 70，營收下降

- 問題：為什麼步入市場後，某項指標反而從 82 降到 70，且營收下降？
- 備註：目前需要先釐清 `82` 與 `70` 分別代表哪個指標，才有辦法正確分析。
- 可追問方向：
  - 是轉換率、毛利率、使用留存、回購率，還是其他營運數字？
  - 是產品進入市場後定價不符、教育成本高，還是通路效率不佳？
  - 是否因客製化成本過高，導致營收與擴張速度受限？

### 肌力改變後是否仍適用

- 問題：肌力是會改變的，改變後原本客製化結果是否仍然適用？
- 可追問方向：
  - 產品是否需要定期重新量測與重新配置？
  - 是否有更新週期、調整機制或版本升級方案？
  - 如果使用者訓練程度提升，原本模型是否會失準？
