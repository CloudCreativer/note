# 022 艾斯冰殼股份有限公司 動態攻擊防護

## 一句話摘要

艾斯冰殼股份有限公司的主題聚焦於動態攻擊防護與應用安全，當前重點在釐清馬來西亞海外擴張理由、APP 開源下合作必要性、POC 驗證方法、動態防護市場量化、與市售方案差異、625 萬資本額下法人股東結構，以及 Android/iOS 支援與 LLM 安全定義與破解難度。

## 背景

- 主題公司：艾斯冰殼股份有限公司
- 主題方向：動態攻擊防護、APP 安全、LLM 相關安全議題
- 已知線索：提及馬來西亞擴張、POC、625 萬與多法人股東、行動平台支援
- 現階段目標：建立技術可信度、商業化路徑與資本結構理解

## 重點整理

### 產品與技術方向

- 動態攻擊防護
- APP 安全（可能含開源組件治理、執行期保護、偵測與回應）
- LLM 安全或模型保護相關能力（定義待確認）

### 初步觀察

- 「開源 APP 為何仍要合作」核心在落地能力、合規、維運與責任承擔
- POC 若缺少嚴謹測試設計，無法作為商業採用證據
- 動態防護商機需區分產業（金融、電商、遊戲、政府）與法規驅動強度
- 小資本額搭配多法人股東，可能反映策略投資或特殊股權安排，需釐清

## 關鍵結論

- 這份筆記目前屬於資安產品驗證與商業模式問題整理階段
- 後續應優先補齊 POC 方法、量化指標與競品比較框架
- 海外擴張與股東結構需對應公司成長策略，避免只看單點資訊
- Android/iOS 與 LLM 安全範圍定義是技術邊界關鍵

## Q&A

### 為什麼馬來西亞作為海外擴張點

- 問題：為什麼馬來西亞是海外擴張起點？
- 可追問方向：
  - 是因為法規、成本、語言、客戶資源，還是合作夥伴關係？
  - 馬來西亞市場的資安需求與台灣相比有何差異？
  - 是區域跳板（東南亞總部）還是單一市場機會？
  - 當地已有 PoC/付費客戶嗎？

### APP 開源，為什麼一定要跟你們合作

- 問題：APP 開源了，為什麼還要跟你們合作？
- 可追問方向：
  - 公司提供的是開源外的專有防護、服務、維運與 SLA？
  - 能否承擔企業級導入（合規、稽核、事件回應、法遵文件）？
  - 開源方案與商業版的功能與風險差異是什麼？
  - 客戶自己維運成本是否高於委外合作？

### POC 是否驗證、怎麼測

- 問題：POC 是否經過驗證？如何測試？
- 可追問方向：
  - 測試情境是否涵蓋真實攻擊路徑（replay、hook、tamper、reverse）？
  - 指標是攔截率、誤報率、延遲影響、穩定性，還是其他？
  - 測試由內部、第三方，還是客戶共同驗證？
  - 是否有可公開測試報告與前後對照結果？

### 動態攻擊防護是哪一類、商機多大、可否量化

- 問題：動態攻擊防護屬於哪一類？商機多大？可否量化？
- 可追問方向：
  - 屬於 RASP、Runtime Protection、Bot/Fraud 防護，還是混合型方案？
  - 目標客群是金融、電商、遊戲、政府，還是一般企業？
  - 市場規模可用 TAM/SAM/SOM 或潛在客戶數量量化嗎？
  - ROI 可否量化（降低詐損、降低資安事故、降低維運成本）？

### 與現有市售功能相比的優勢

- 問題：與現有市售功能相比，優勢是什麼？
- 可追問方向：
  - 優勢在檢測精度、效能開銷、部署速度、價格，還是整合彈性？
  - 可支援的攻擊場景是否比競品更完整？
  - 與主流競品（本地或國際）差異化的量化指標為何？
  - 是否具備客戶續約率或轉單案例可佐證？

### 625 萬為何有那麼多法人股東

- 問題：資本額 625 萬，為什麼會有那麼多法人股東？
- 可追問方向：
  - 法人股東是策略合作、財務投資，還是資源交換型股東？
  - 股權是否過度分散，是否影響決策效率？
  - 是否有特別股、表決權安排或可轉換條件？
  - 這些法人是否實際帶來客戶、技術或通路資源？

### Android / iOS 支援、LLM 定義與破解難度

- 問題：Android、iOS 是否有支援？LLM 定義與現在破解難度？
- 可追問方向：
  - 目前支援範圍是 SDK、API、Agent 還是全端防護？
  - Android/iOS 是否有同等防護深度與維運能力？
  - LLM 安全定義是 prompt injection、防資料外洩、模型竄改，還是帳密濫用？
  - 所謂破解難度是攻擊成本提升、時間延長，還是可偵測率提升？
