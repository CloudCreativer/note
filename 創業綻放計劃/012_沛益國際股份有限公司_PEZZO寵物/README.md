# 012 PEZZO 寵物

## 一句話摘要

PEZZO 的主題聚焦於寵物科技，結合寵物臉部辨識、醫療紀錄與保險／SaaS 服務，從 B2B 延伸至 B2C，目標是保障飼主、降低產險公司理賠負擔並串接產業鏈；目前仍需釐清訂閱制必要性、專利與東麗科技關係、股權資料差異，以及貓狗以外品類與金融機構角色。

## 背景

- 主題公司：PEZZO
- 主題方向：寵物（臉部辨識、病例管理、保險、產業鏈串接）
- 已知合作／資本線索：骨感集團入資、複數表決權約 74%；專利與東麗科技（12/26，細節待查）
- 現階段目標：整理現場 Q&A 與已知 Ans，並標記待查證項目

## 重點整理

### 產品與商業方向

- 寵物臉部辨識：計算臉部特徵值，確保寵物相似度，結合晶片、醫療紀錄
- APP：歸納寵物病例；SaaS 有免費與收費方案；串接保險
- 商業路徑：B2B → B2C（「拉出來」— 具體分工待釐清）
- 價值主張：保障飼主、降低產險公司回覆／理賠率、串接產業鏈

### 初步觀察

- 技術核心在「身份辨識 + 醫療資料 + 保險」，而非單純 C 端 APP
- 日本市場與寵物保險法規高度相關，不能只用「保險額高」單一因素解釋
- 股權結構（創辦人簡報 43.92% vs 公開查詢、骨感集團 74% 表決權）需交叉驗證
- 專利日期 12/26 與東麗科技關係需查專利庫與授權文件

## 關鍵結論

- 競爭優勢敘事偏向產業鏈整合（辨識 + 病例 + 保險 + 產險效率）
- 訂閱制可能服務於 SaaS／保險生態，而非唯一收入來源
- 日本策略需結合「國家規定」與保險產品設計一併理解
- 投資人應優先核對：專利歸屬、股權／表決權、創辦人持股公開資料差異

## Q&A

### 訂閱制的必要性

- 問題：訂閱制的必要性是什麼？
- 已知答覆（部分）：SaaS 區分免費與收費，並與保險場景結合。
- 可追問方向：
  - 訂閱解鎖哪些功能（進階病例、理賠、多寵管理）？
  - 若保險為主收入，訂閱是否只是獲客或數據入口？
  - 免費版與付費版的轉換率假設？

### 寵物多元如何精準

- 問題：寵物種類多元，如何做到精準？
- 已知答覆：以臉部辨識計算特徵值確保相似度，並結合晶片、醫療紀錄。
- 可追問方向：
  - 貓、狗以外品類（兔、鳥、爬蟲等）是否支援？準確率為何？
  - 同品種相似外觀的誤判率與校正機制？
  - 晶片 ID 與臉部辨識衝突時以誰為準？

### 專利 12/26 與東麗科技

- 問題：專利與 12/26、東麗科技的關係為何？
- 備註：使用者筆記「專利是 12/26 東麗科技」，需查證為申請日、公告日或合約日期。
- 可追問方向：
  - 專利權人為 PEZZO、東麗科技，還是共同持有？
  - 是技術授權、材料供應，還是共同研發？
  - 對台灣／日本市場是否有地域或用途限制？

### 投資額與股權差異（43.92% vs 查詢結果）

- 問題：創辦人投影片 43.92% 與實際查詢為何不同？
- 已知答覆（部分）：骨感集團入資後，複數表決權約 74%。
- 可追問方向：
  - 43.92% 是指創辦人個人、創辦團隊，還是某輪投資前後？
  - 74% 表決權來自特別股、投票協議或董事席？
  - 公開查詢（商工登記、投資揭露）的基準日與投影片是否一致？

### 日本寵物保險與保險額

- 問題：日本寵物保險是否因為保險額高而成立？
- 已知答覆：與日本國家規定有關（現場說法）；使用者原問題亦提到「保險額高」。
- 可追問方向：
  - 具體是哪類法規（強制投保、動物登記、獸醫紀錄）？
  - 台灣與日本產品設計差異？
  - 保險額、自負額、理賠流程如何與臉部辨識／病例串接？

### 競爭優勢

- 問題：如何建立競爭優勢？
- 已知答覆：保障飼主、降低產險公司回覆率、串接產業鏈。
- 可追問方向：
  - 相對其他寵物保險或 Pet tech 的量化優勢（理賠時間、詐保率、成本）？
  - 產險公司合作是否獨家或排他？
  - 數據護城河（病例量、辨識模型）規模為何？

### 金融機構、飼主、貓狗以外品類

- 問題：客群是金融機構、飼主，還是包含貓狗以外的寵物？
- 已知答覆（部分）：B2B → B2C；產品面向飼主與產險／產業鏈。
- 可追問方向：
  - B2B 端是產險、獸醫院、寵物店，還是金融機構？
  - 「金融機構」角色是資金方、保險共保，還是支付／分期？
  - 路線圖是否支援多物種保險 SKU？
