# 004 睿泉生技 製藥AI-agents

## 一句話摘要

睿泉生技的主題聚焦於製藥產業中的 AI agents 應用，目前重點在釐清製程數據來源、AIX Agent 的收費模式、end-to-end 收費邏輯，以及整體商業模式究竟偏向 AI 產品、SaaS 平台，還是顧問服務。

## 背景

- 主題公司：睿泉生技
- 主題方向：製藥產業、AI agents
- 已知切入點包含製程數據、AIX Agent、end-to-end 收費模式
- 現階段目標是建立產品定位理解，並整理資料來源與商業模式相關問題

## 重點整理

### 產品與技術方向

- 製藥產業應用
- AI agents
- 製程數據分析
- 可能涉及 end-to-end 工作流程整合

### 初步觀察

- 若產品面向製藥產業，核心價值可能來自流程優化、知識整合、決策輔助或自動化分析
- 若提到 AIX Agent，需先確認這是產品正式名稱、平台模組名稱，還是對外描述用語
- 製藥場景對資料可信度、可追溯性與導入門檻通常較高，因此資料來源與收費方式會直接影響商業模式成立性
- 若基礎技術從 end-to-end 都收費，代表公司可能不是只賣單點工具，而是主張整體流程價值

## 關鍵結論

- 這份筆記目前屬於產品定位與商業模式梳理階段
- 後續應優先補齊製程數據來源、產品定義與實際收費模式
- 若要判斷公司護城河，需特別釐清其在製藥流程中的資料掌握與整合能力

## Q&A

### 製程數據從哪裡來

- 問題：製程數據從哪裡來？
- 可追問方向：
  - 來自藥廠內部資料、合作案資料、實驗數據，還是公開資料？
  - 資料是即時串接、歷史資料匯入，還是由客戶手動提供？
  - 在製藥場景下，資料取得是否涉及法規、隱私或權限限制？

### AIX Agent 收費模式

- 問題：AIX Agent 的收費模式是什麼？
- 可追問方向：
  - 是按帳號數、使用量、模組數、案場數，還是專案制收費？
  - 是訂閱制、一次性授權費，還是平台加顧問導入費？
  - 是否區分 PoC、正式導入與長期維運的不同價格層級？

### 為什麼基礎技術要從 end-to-end 都收費

- 問題：基礎技術為什麼要從 end-to-end 全段收費？
- 可追問方向：
  - 是因為價值不在單一模型，而在整體流程整合、驗證與落地？
  - 是否每一段流程都需要客製化、資料治理與系統串接，因此成本高？
  - 若只賣單點功能是否不足以支撐效果，才需要整段一起打包？

### 商業模式是提供 AI、SaaS 還是顧問

- 問題：商業模式究竟是提供 AI、SaaS，還是顧問服務？
- 可追問方向：
  - 是否有標準化平台，可重複複製到不同藥廠或製程場景？
  - 收入主要來自軟體授權、訂閱費、客製案收入，還是顧問與導入服務？
  - 若客戶導入高度依賴專案客製，則 SaaS 化程度可能有限
