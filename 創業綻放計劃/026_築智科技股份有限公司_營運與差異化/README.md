# 026 築智科技股份有限公司 · 營運與差異化

## 30 秒看懂

築智（`SparkGen`）主打「營造業垂直 AI」，核心是以幾何模型與 BIM/法規語意引擎打通估算、設計到工地資訊流。  
簡報已給出產品下載/企業客戶、現有訂單金額、競品比較、TAM/SAM/SOM 與三年 GTM/財務預估，屬於可量化的擴張敘事。

## 他們在做什麼

- 產品方向（逐字稿口述）：以 AI 工具解決現場估算、排程、溝通與交付效率問題。
- 客群輪廓：工程/營建鏈為主，並延伸到其他具流程標準化需求的產業。
- 技術敘事：強調在地工法/法規適配與客製化能力，而非純通用模型。

## 解決什麼問題

- 傳統工程流程知識分散，估算與協作高度仰賴個人經驗，效率不穩。
- 多國/多場域雖有同類工具，但在地法規、工法與語言差異造成落地困難。
- 用戶常以共用帳號降低成本，導致資安與商業模式風險。
- 新創在快速擴張時容易同時做太多市場，造成資源稀釋。

## 怎麼賺錢

- 商業模式：專案導入 + 客製服務 + 逐步產品化。
- 客單來源：來自既有產業關係與轉介紹，並開始往更標準化方案轉。
- 中期目標：提高可重複收入比重，降低一次性專案依賴。
- 海外策略：先做可複製場景，再拓展到美國市場合作機會。

## 實績與進展（決賽現場）

- 逐字稿可辨識主張：
  - 產品 traction：`LandGen 80+`（另一頁標示 `75+`）專業用戶下載、`DesignGen 3` 家企業訂閱。
  - 訂單與案型：簡報列出 `200萬 / 400萬 / 600萬/年` 等案型，並揭露預鑄切圖已簽約 `1200~1800萬/年`。
  - 市場規模頁揭露：`SOM 3.53億/年`、`SAM 35.3億/年`、`TAM 480億/年（15億美金）`。
- 評審追問焦點：
  - 與美國現有工具相比，真正差異在哪。
  - 帳號共用問題是否已有可落地防堵機制。
  - 未來一到兩年究竟要聚焦哪些產業/模組。
  - 共享辦公室營運型態是否足以支撐規模化。

## 評審提問與回答

### Q1. 跟美國同類工具相比，你們差異是什麼？
- 團隊回答：差異在在地工法/法規適配、可客製能力與現場落地效率，海外工具多為通用邏輯。
- 正確性註記：`中信心`
  - 理由：敘事合理，但缺量化對照數據。

### Q2. 一個帳號多人共用怎麼解？
- 團隊回答：現況確有共用情形，團隊正在優化機制與方案設計，降低共用誘因。
- 正確性註記：`中高信心`
  - 理由：回覆誠實且與評審追問一致；但具體機制尚未完整公開。

### Q3. 未來一到兩年你們到底聚焦哪裡？
- 團隊回答：先聚焦高痛點場景，把可複製引擎做穩，再擴到其他產業與海外。
- 正確性註記：`中信心`
  - 理由：方向清楚，仍缺量化里程碑（ARR、客戶數、續約率）。

### Q4. 公司登記在共享辦公室，營運是否需要實體空間？
- 團隊回答：目前可借用合作場域運作，後續隨成長搬遷至更完整辦公空間。
- 正確性註記：`中高信心`
  - 理由：與新創早期營運常態一致，敘事自洽。

## 我學到的 3 件事

1. 競品差異若不能量化，評審通常不會買單。
2. 帳號治理是 SaaS 商業化的基本功，不只是資安議題。
3. 早期新創用共享辦公室合理，但擴張期需同步規劃組織承載能力。
