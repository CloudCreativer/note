# 021 蘊域科技 · EVS 跌倒偵測

## 30 秒看懂

蘊域科技（決賽主持／逐字稿作「音樂科技」）用 **EVS（事件式視覺）** 做**跌倒與異常行為偵測**——只記錄畫面**變化**，非連續錄影，隱私友善（GDPR／HIPAA 方向）。  
商業路徑：**硬體 EVS5（克羅諾）** 打機構累積資料 → **SDK** 嵌入他廠設備收授權費；已簽**新光保全**等，2026 營收目標約 **430 萬美元**。

---

## 他們在做什麼

| 面向 | 說明 |
|------|------|
| **技術** | EVS 感測（合作 **Prophesee**、聯發科等）；非 CMOS 連拍，為**光折射**事件成像 |
| **產品** | **EVS5** 硬體；**SDK** 行為演算法授權 |
| **場景** | 長照、醫療機構、居家（像 iCam 安裝） |
| **資料** | 宣稱全球最大規模之一之 **EVS 人類行為資料庫**（約六年；早期與 Prophesee 獨家） |

---

## 解決什麼問題

- 長照／醫療：傳統攝影隱私疑慮、雷達易受環境干擾；穿戴式老人不願戴。  
- EVS：黑底點雲、**毫秒級**反應（簡報稱跌倒場景約 **97.5%** 醫療級水準敘事）。  
- 機構需即時通報；居家需 Wi‑Fi 通報多台手機。

---

## 怎麼賺錢

```text
Phase 1：EVS5 硬體 → 機構場域 + 真實數據
Phase 2：SDK 授權 → 智慧家庭／設備大廠嵌入
```

| 收入 | 說明 |

**募資用途（簡報）：** 次世代裝置、核心專利、晶片量產、海外展會等。

### 通報場景（Q&A）

- **居家：** 裝機 → Wi‑Fi 對接手機，一台通報多台。  
- **機構：** 納入照護系統（MQTT 等）。  
- **新光：** 納入 SKS 系統。

### 1 ms 說明（Q&A）

- 技術上幀率可達極高（現場稱最高約 **10,000 fps** 量級）；跌倒場景實際約 **62 fps**。

## 評審提問與回答

### Q1. 專利不好申請，很多單位要用會不會很快有競爭者？

我們有約六年、全球最大規模之一的 **EVS 人類行為資料庫**（早期與 Prophesee 獨家）。Sony、Prophesee 做硬體，我們做場域演算法，是合作關係。台灣在獨家期內幾乎無同技術競爭者；大廠從談判到產品至少約**三年**時間差。

### Q2. 賽道領先者之外，還有沒有其他競爭者？

同上。Sony 近兩年積極推 EVS，技術來源亦與 Prophesee 相關。

### Q3. 有鎖定個人家用或機構市場嗎？

全台醫療／長照示範與出貨約 **50+**；產品可裝居家，亦主攻機構。

### Q4. 物理感測是用 CMOS 還是不同原理？

**光折射**成像，只記錄變化，非 CMOS 連續攝影；符合 GDPR／HIPAA 方向（現場說法）。

### Q5. 能否用 30 秒說明從跌倒到通報的場景？

居家：像 iCam 裝機、Wi‑Fi 對接手機即可；機構：訊號納入照護系統；新光：納入 SKS。

### Q6. 行政管理、人員配置與法遵？

曾發生勞健保轉換過程錯誤，**已釐清**（現場簡短說明）。

### Q7. 反應速度 1 ms 如何定義？

技術上可達極高幀率；人類行為感測跌倒約用 **62 fps**（虛擬鍵盤等應用約 2.5 fps）。

## 我學到的 3 件事

1. **隱私友善是 EVS 的第一個商業切角**：先解決攝影監控在長照/醫療場域的隱私障礙，才能進入部署決策。  
2. **資料庫與場域部署是護城河核心**：硬體感測器可被替代，但長期資料與客製化演算法更難複製。  
3. **B2B 策略會從硬體走向授權**：先靠裝置落地拿到資料與客戶，再以 SDK 擴張到更多設備與通路。
