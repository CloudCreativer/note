## E1 名單轉換 SOP

### YV-001｜SOP 流程引擎（狀態機）

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | 核心 |
| **描述** | 對話依狀態推進，非單次 QA；狀態可配置、可分支 |
| **驗收標準** | 後台可視化流程：節點、條件、下一狀態 |
| **備註** | 與「知識庫一次性回覆」根本不同 |

### YV-002｜Stage 0：開場（非個資）

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | SOP |
| **描述** | 進線後先閒聊、情緒鋪墊；**禁止**第一則就問電話 |
| **驗收標準** | 前 N 輪無個資欄位卡片 |
| **依賴** | E2 情緒 Prompt |

### YV-003｜Stage 1：需求探詢

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | SOP |
| **描述** | 釐清：想上課？交女友？交男友？認識人？ |
| **驗收標準** | 需求寫入 user profile，影響後續話術 |

### YV-004｜Stage 2：性別確認

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | SOP |
| **描述** | 早期確認男／女（頭像常误导） |
| **驗收標準** | 性別未確認前不推異性向錯誤內容 |
| **備註** | 客戶：「聊半天才發現是女生」 |

### YV-005｜Stage 3：稱呼收集

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | SOP |
| **描述** | 自然問稱呼：「請問方便稱呼您為王先生嗎？」 |
| **驗收標準** | 儲存 display_name，供後續人工致電使用 |

### YV-006｜Stage 4：名單欄位卡片

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | SOP／訊息 |

| **驗收標準** | 五欄可分段或一次；資料寫入 CRM |
| **依賴** | YV-002～005 完成後才可觸發 |

### YV-007｜資格篩選規則引擎

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | 規則 |
| **描述** | 自動篩：未成年拒絕、>45 歲拒絕、無業可標記為低優先 |
| **驗收標準** | 不符資格時禮貌結束＋不分配顧問 |
| **備註** | 規則需後台可調 |

### YV-008｜區域分配顧問

| 欄位 | 內容 |
|------|------|

| **驗收標準** | 名單匯出或通知時含分配結果 |
| **依賴** | YV-006 |

### YV-009｜Step 流失偵測

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | 流程 |

| **驗收標準** | 後台列表：卡在哪一步、多久未回 |
| **備註** | 客戶：「第一步、第二步、第三步就不回了」 |

### YV-010｜流失 → 人工介入佇列

| 欄位 | 內容 |
|------|------|

### YV-012｜「立即開始配對」入口改寫

## E2 對話體驗

### YV-020｜情緒價值 Tone Profile（交友／閨蜜感）

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | AI／Prompt |
| **描述** | ChatGPT 式：貼心、會共情、「戳中」對方；避免制式客服 |
| **驗收標準** | 盲測：客戶團隊評分優於現有自動回覆 |

### YV-021｜半自動模式：AI 草稿＋人工發送

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | 後台 |
| **描述** | AI 生成建議回覆，客服可改字再送（過渡方案） |
| **驗收標準** | 訓練期可用；修正紀錄可餵回模型 |

### YV-022｜人工純手打通道

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | 後台 |
| **描述** | 客服在後台直接打字，用戶端不顯示「AI 標籤」 |
| **驗收標準** | 與 AI 訊息混在同一 thread，用戶感受真人 |
| **備註** | 「下面開始打字就是我人工打的」 |

### YV-023｜AI／人工訊息視覺不區分

| 欄位 | 內容 |
|------|------|
| **優先級** | P0 |
| **類型** | UX |
| **描述** | LINE 端不標示 bot；避免用戶發現「從頭到尾都是 AI」 |
| **驗收標準** | 無 bot badge 或制式 footer |

### YV-024｜重複制式回覆偵測

| 欄位 | 內容 |
|------|------|
| **優先級** | P1 |
| **類型** | AI |
| **描述** | 同一用戶收到相同模板多次 → 自動改寫或轉人工 |
| **驗收標準** | 降低「按某東西一直出現相同回覆」的 AI 感 |
| **備註** | 誤觸卡片也會觸發此問題 |

### YV-025｜夜間模式（21:00 後）

| 欄位 | 內容 |
|------|------|
| **優先級** | P1 |
| **類型** | 規則 |
| **描述** | 非緊急可延後；AI 可先回情緒安慰；急件標記人工 morning |
| **驗收標準** | 後台可設時段策略 |

### YV-026｜情緒上頭用戶優先

| 欄位 | 內容 |
|------|------|
| **優先級** | P1 |
| **類型** | AI |
| **描述** | 偵測孤單、負面情緒 → 優先安慰，暫緩推銷／要電話 |
| **驗收標準** | 付費線、免費線皆可觸發 |

### YV-027｜無禮貌用戶冷處理

| 欄位 | 內容 |
|------|------|
| **優先級** | P2 |
| **類型** | 規則 |
| **描述** | 偵測辱罵／騷擾 → 降速回覆或隔日再回 |
| **驗收標準** | 可人工覆寫 |

### YV-028｜預設回覆腳本庫（5～35 條）

| 欄位 | 內容 |
|------|------|
| **優先級** | P1 |
| **類型** | 內容 |
| **描述** | 客戶現有「自己打的」腳本匯入，供 AI 參考或一鍵插入 |
| **驗收標準** | 至少 5 條可在後台維護 |

## E3 LINE UX

### YV-030｜Flex 卡片觸控區縮小

### YV-034｜卡片與純文字混用策略
