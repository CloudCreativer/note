# 學習筆記架構

> Google 數位人才探索計劃 · 把「聽過」變成「學會」

---

## 為什麼需要這套架構

知識如果只停留在「看過」或「聽過」，通常很快會流失。  
這套架構的目的，是強迫自己完成三件事：

1. **用自己的話說一遍** — 確認真的理解，而不是記住原文
2. **說清楚為什麼重要** — 產生認知升級，而不只是收藏資訊
3. **轉成行動** — 明天就能做一點什麼，知識才會留下來

---

## 完整版（六欄位）

適合第一次接觸某個概念、需要深度消化時使用。

| 欄位 | 英文 | 回答什麼 | 提示 |
|------|------|----------|------|
| 核心觀點 | What | 今天學到什麼？ | 1～3 句，能濃縮就代表真的吸收了 |
| 原始內容 | Input | 原本看到或聽到什麼？ | 不用寫太多，重點是留下來源 |
| 我的理解 | Translation | 用自己的語言重新解釋 | **最重要**。寫不出來通常代表還沒學會 |
| 為什麼重要 | Why | 這改變了我什麼想法？ | 以前覺得 → 現在覺得 |
| 如何應用 | Action | 明天就能做什麼？ | 沒有行動，知識通常會快速流失 |
| 延伸問題 | Question | 還有哪些疑問？ | 好的問題會帶來下一輪學習 |

### 各欄位範例

**核心觀點（What）**

> AI 不會取代人，而是會取代不會使用 AI 的人。

> 投資人投的不是技術，而是市場需求。

**原始內容（Input）**

- 來源：課程 / 演講 / 書籍 / Podcast / 對話
- 記錄：演講者說什麼、書上寫什麼、課程提到什麼

**我的理解（Translation）**

| 原文 | 我的理解 |
|------|----------|
| Product-Market Fit 是找到市場需求。 | PMF 其實是在證明你解決的是大家願意付錢的問題，而不是自己覺得重要的問題。 |

**為什麼重要（Why）**

| 以前覺得 | 現在覺得 |
|----------|----------|
| 功能做得夠好就會有人用。 | 需求比功能更重要。 |

**如何應用（Action）**

- 學到：Pitch 要先講痛點
- 行動：修改 Cloudia Pitch 第一頁 — 先講客戶問題，再講 AI 功能

**延伸問題（Question）**

- 為什麼有些產品沒有 PMF 還能活下來？
- 投資人如何驗證市場需求？
- Cloudia 現在真的有 PMF 嗎？

## 極簡版（五欄位）★ 日常使用

日常學習優先用這版。夠快、夠深、夠用。

| 欄位 | 內容 |
|------|------|
| **Input** | 聽到什麼 |
| **Insight** | 我學到什麼 |
| **Why** | 為什麼重要 |
| **Action** | 我要做什麼 |

## 填寫檢查清單

寫完一篇筆記後，快速自問：

- [ ] **Insight** 能在 30 秒內講給別人聽嗎？
- [ ] **Input** 有標明來源，之後找得到嗎？
- [ ] **Translation / Insight** 是自己的話，不是複製原文嗎？
- [ ] **Why** 有寫出「以前 → 現在」的認知變化嗎？
- [ ] **Action** 是明天就能做的一小步嗎？
- [ ] **Question** 至少有一個值得繼續追的問題嗎？

若 **Translation** 寫不出來 → 代表還沒真的學會，回去重聽或重讀。  
若 **Action** 寫不出來 → 知識還停在「知道」，尚未進入「會用」。
