# 學習筆記架構

> 線上與實體課程 · 把「聽過」變成「學會」

---

## 為什麼需要這套架構

知識如果只停留在「看過」或「聽過」，通常很快會流失。  
這套架構的目的，是強迫自己完成三件事：

1. **用自己的話說一遍** — 確認真的理解，而不是記住原文
2. **說清楚為什麼重要** — 產生認知升級，而不只是收藏資訊
3. **轉成行動** — 明天就能做一點什麼，知識才會留下來

---

## 極簡版（五欄位）★ 日常使用

| 欄位 | 內容 |
|------|------|
| **Input** | 聽到什麼 |
| **Insight** | 我學到什麼 |
| **Why** | 為什麼重要 |
| **Action** | 我要做什麼 |
| **Question** | 還有哪些疑問 |

## 含圖片筆記

若 `assets/images/` 有圖或舊稿含 `![...]`：

1. 逐張辨識（OCR／視覺理解）
2. 將圖中重點併入正文（表格、條列、數字）
3. `has_images: true`，並在文末附「圖片對照」表

## 填寫檢查清單

- [ ] **Insight** 能在 30 秒內講給別人聽嗎？
- [ ] **Input** 有標明來源與附件路徑嗎？
- [ ] **Translation** 是自己的話嗎？
- [ ] **Why** 有「以前 → 現在」嗎？
- [ ] **Action** 是明天就能做的一小步嗎？
- [ ] 含圖時，不看圖也能理解八成內容嗎？
